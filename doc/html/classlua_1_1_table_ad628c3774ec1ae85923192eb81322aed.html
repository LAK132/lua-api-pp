Create a new empty table.

Due to some special circuimstances*, it is possible to create anonymous tables with this constructor to use in function calls and generally to create any Temporary object, but you can't use it for permanent objects. So if you want to create anchor object, or push a new table into value set, use records or array function instead.

Parameters
arrSizeAmount of array entries (numeric-indexec) to preallocate
recSizeAmount of record entries (arbitrary indices) to preallocate
Note
*Anonymous table is created before the initializers for temporary are evaluated. Then it is copied by reference to initialize the temporary object. After the temporary object is used, discarded and removed from the stack, anonymous table is destroyed and also removed from the stack.
A permanent object created after the anonymous table would outlive it and thus break the stack integrity.
