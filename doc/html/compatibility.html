Compatibility

The library supports several Lua versions listed below. By default the library selects the most recent API version known to it.

Lua 5.1

Also compatible with LuaJIT. Turned on with LUAPP_COMPATIBILITY_V51 macro.

Differences to 5.2 mode:

  • natural arithmetics is not supported;
  • length operation returns native size_t instead of producing Lua value;
  • the gcIsRunning function is not available;
  • the getVersion function is not available;
  • the raw accessor does not use specialized pointer get/set functions.

Lua 5.2

Turned on by default ro with LUAPP_COMPATIBILITY_V52.
